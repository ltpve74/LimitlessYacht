# AGENTS.md — cross-agent guide for this repo

**Vendor-neutral entry point** for any AI coding agent (Claude Code, Codex, Cursor, Windsurf, Aider,
Copilot, goose, …) working on the Limitless Yacht site. The full, canonical rules live in
**[CLAUDE.md](CLAUDE.md)** (workflow) and **[DECISIONS.md](DECISIONS.md)** (why the non-obvious
choices exist) — read those before non-trivial work. This file is a short index of the rules an
agent **must** know so it doesn't break the release process or undo a load-bearing decision.

## Agent workspace — nothing agent-related ships to production

All agent working files live in **`.agent/`** (see [`.agent/README.md`](.agent/README.md)). It is
**stripped from `main`** on publish, so nothing there reaches production. **Put new working files,
notes, and memory there — not at the repo root.**

- **Shared cross-agent memory:** [`.agent/memory/`](.agent/memory/) — markdown facts (one per file)
  + a `MEMORY.md` index. Read and update it so knowledge carries across agents.
- **Briefs / handovers / scratch:** `.agent/briefs/`, or anywhere under `.agent/`.
- These four instruction files stay at the repo **root** so agents auto-load them, but they are
  **also stripped from `main`**: `CLAUDE.md`, `AGENTS.md`, `DECISIONS.md`, `HANDOVER.md`.

## Branch flow & publishing

- **`develop`** = readable source of truth (multi-line HTML/CSS). Do all work here (or a feature
  branch off it). **`main`** = production, **minified** (single-line). Netlify deploys `main`.
- **Never edit HTML/CSS source directly on `main`** — always merge from `develop`.
- **Never back-merge `main` → `develop`** (that pulls minified files into the readable branch).
- **`develop` showing "N commits behind `main`" is NORMAL.** Publishes are `main`-only minified
  commits; `develop` still holds all the source. Don't "fix" this.
- **Publishing** = merge `develop` → `main` (dev wins on conflicts), commit `"Publish: …"`. The
  `main` pre-commit hook then **minifies + runs the publish gate** (site tests → UX smoke →
  Lighthouse budgets). This is **slow by design** (~a minute+) — a long publish is the gate running,
  not a hang. Don't re-analyze; let it finish.
- **Don't hand-edit `de/ es/ fr/`** — they're generated by `i18n/build-locales.py`. New/changed
  visible English strings need matching PAIRS in **all three** `i18n/locales/{de,es,fr}.py`.

## Load-bearing decisions — DO NOT "fix" without reading DECISIONS.md

These look like easy wins but are deliberate, measured, and **test-guarded** (`# DECISION` tests +
a pre-commit lock; weakening a guard to make a change pass is blocked):

- **Montserrat font is OFF the critical path** — there is **no `@font-face`** for it in `main.css`
  or the critical block, and no `<link rel=preload>` for it. A reported "font shift" is **not** a
  reason to revert — the fallback is metric-matched; **measure CLS first** (it's ~0.01, no shift).
- **The net-tier boot loader stays INLINED**, not an external `<script src>` (avoids a
  render-blocking fetch ahead of all the CSS).
- **`main.css` loads ~300 ms after `layout.css`** — protects the hero LCP; don't parallelize or
  restore the old 1500 ms defer.
- **Keep the CLS reserves** (reviews grid, hero pull-quote margin, carousel-nav min-height).
- **Critical CSS** must use `var()`-free literals and plain class-list selectors (minifier-safe);
  budget is ~13.5 KB.

## Repo hygiene

- **`screenshots/` NEVER goes to `main`/production.** It lives on `develop`/feature branches for
  visual feedback and is stripped on publish (also excluded from Netlify + GitHub Pages). Do **not**
  restore or re-commit its deletions — it's actively managed. Read screenshots as feedback input only.
- **No auto-push:** commit locally; push only when the human agrees.
- The owner's **media library** (`media-library/`) holds originals — never commit raw
  originals/video; the repo carries only processed, site-ready derivatives (via
  `scripts/process_media.py`).

## Where things live

| What | Where |
|------|-------|
| Pages (edit) | `index.html`, `legal.html` |
| Styles (edit) | `css/main.css`, `css/layout.css` |
